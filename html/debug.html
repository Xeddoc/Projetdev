Debug

Optimisation de l'utilisation du port série en mode DEBUG

La gestion du port série demande des ressources, donc si on n'est pas en mode DEBUG, on peut s'en passer. Ainsi tous les appels direct au port série de type Serial.print() ou Serial.println() seront remplacés par l'utilisation de macros.

Si le mode debug est activé alors les macros seront remplacées par des appels au port série; sinon ils seront juste ignorés.

Note
L'activation du mode debug se fait à l'aide de MYDEBUG. Il faut donc qu'il soit initialisé avant le #include de ce fichier si on veut qu'il soit pris en compte.
